# Security Policy

This document outlines how to report vulnerabilities and which versions of the project are actively supported with security updates.

## Supported Versions

Only the latest major/minor releases are actively maintained and receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.6.x   | :white_check_mark: |
| < 1.6.0 | :x:                |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability in this project, please report it privately to the maintainers:

* Email: **[jesusoteromartinez@outlook.com](mailto:jesusoteromartinez@outlook.com)**

Please include the following details in your report:
* **Description:** A detailed explanation of the vulnerability and its potential impact.
* **Steps to Reproduce:** A step-by-step guide (including scripts or mock data, if possible) to trigger the issue.
* **Environment details:** The OS, database state, API version, or client configuration tested.

### Our Response Process

1. **Acknowledgement:** You will receive an initial email receipt acknowledgement within **48 hours**.
2. **Assessment:** We will triage the report and verify the vulnerability.
3. **Fix and Disclosure:** If confirmed, we will coordinate a patch. We aim to resolve reported vulnerabilities within **15 days** of initial triage, followed by a coordinated public security advisory.
